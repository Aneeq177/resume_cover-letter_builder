# CrewAI Resume Builder

An agentic pipeline that takes a resume and a job posting and produces a tailored, ATS-optimised resume and cover letter — ready to send.

## How it works

Five specialised agents run in sequence:

| # | Agent | Model | Job |
|---|-------|-------|-----|
| 1 | Resume Parser | Haiku | Extracts every field from your resume into structured JSON |
| 2 | Job Analyst | Haiku | Scrapes the JD, extracts ATS keywords, researches the company |
| 3 | Resume Tailor | Haiku | Rewrites and reorders the resume for ATS + recruiter fit |
| 4 | Cover Letter Writer | Haiku | Writes a targeted, company-specific cover letter |
| 5 | QA Reviewer | Haiku | Audits both documents, applies fixes, scores the package |

### Key behaviours

- **Student / entry-level detection** — if the candidate has < 3 years of experience or is currently enrolled, the tailor enforces a strict 1-page output, consolidates thin entries into a "Campus Involvement" line, and trims soft-skill keyword soup.
- **No hallucinations** — statistics and metrics in the cover letter must be traceable to the parsed resume. Any invented figure is removed.
- **Date accuracy** — the cover letter date is injected from the system clock, not generated by the model.
- **ATS scoring** — every draft is scored against extracted JD keywords; the target is ≥ 75% coverage before QA runs.

## Outputs

All files are written to `outputs/` (gitignored — contains personal data):

| File | Contents |
|------|----------|
| `parsed_resume.json` | Structured resume data |
| `job_analysis.json` | JD keywords, company intel, application angle |
| `tailored_resume.md` | Rewritten resume |
| `cover_letter.md` | Cover letter draft |
| `final_resume.md` | QA-approved resume |
| `final_cover_letter.md` | QA-approved cover letter |
| `qa_report.json` | Full audit scorecard with ATS score, verdict, fixes |

## Setup

### 1. Clone and create a virtual environment

```bash
git clone https://github.com/Aneeq177/crewai-resume-builder.git
cd crewai-resume-builder
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure API keys

```bash
cp .env.example .env
# Edit .env and fill in your keys
```

You need three API keys:

| Key | Where to get it |
|-----|----------------|
| `ANTHROPIC_API_KEY` | [console.anthropic.com](https://console.anthropic.com/account/keys) |
| `SERPER_API_KEY` | [serper.dev](https://serper.dev) |
| `VOYAGE_API_KEY` | [voyageai.com](https://www.voyageai.com) |

## Usage

```bash
# Job posting URL
python main.py --resume resume.pdf --job https://jobs.lever.co/acme/123

# Local job description file
python main.py --resume resume.pdf --job job_description.txt

# Pause before QA so you can inspect the draft first
python main.py --resume resume.pdf --job job.txt --human-review
```

Resume formats supported: `.pdf`, `.txt`, `.docx`

## Project structure

```
.
├── main.py          # CLI entrypoint, pipeline wiring, output parser
├── agents.py        # All five CrewAI agent definitions
├── task.py          # All five task prompts with context chaining
├── tools.py         # Custom BaseTool implementations (ATS scorer, consistency checker, etc.)
├── requirements.txt # Python dependencies
├── .env.example     # API key template
└── outputs/         # Generated files (gitignored)
```

## Requirements

- Python 3.10+
- An Anthropic API key (Claude Haiku is used for all agents)
- A Serper API key (web search for company research)
- A Voyage AI key (embeddings)
